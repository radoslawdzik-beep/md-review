Review markdown bundles in a folder

Click Open folder… and pick the folder containing the .md files (e.g. your unzipped review-bundle). The browser will ask permission to read and write in that folder.

How it works

• Originals are never modified. When you edit a file, changes are saved to a NAME.REVIEWED.md copy next to it.
• Edits use CriticMarkup: {++added++}, {--deleted--}, {~~old~>new~~}, {>>comment<<}, {==highlight==} — rendered in colour in Preview, and readable as plain text by anyone.
• Approve / Request changes per file; statuses and comments are saved to .review-state.json in the folder, tagged with the reviewer name.
• ＋ New .md file adds your own files to the bundle.
• Export summary writes REVIEW_SUMMARY.md with all statuses and comments — send that back with the folder.
